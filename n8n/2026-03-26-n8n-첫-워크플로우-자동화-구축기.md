### Context

* 디스코드(Discord) 메시지를 트리거로 삼아 수신된 텍스트를 TIL 형식으로 변환하고, 이를 깃허브(GitHub) 레포지토리에 자동 푸시(Push)하는 워크플로우를 구축하고자 함.
* 깃허브 푸시 이후 자동으로 README.md 파일을 업데이트하여 인덱스를 추가하고, 최종적으로 완료 메시지를 디스코드로 재발송하는 풀 사이클(Full-cycle) 자동화 시스템이 필요했음.

### Core

* n8n 워크플로우의 주요 노드 구성은 다음과 같은 파이프라인으로 설계됨.

```json
{
  "nodes": [
    {
      "parameters": {
        "event": "messageCreate",
        "channelId": "YOUR_DISCORD_CHANNEL_ID"
      },
      "type": "n8n-nodes-base.discordTrigger",
      "name": "Discord Trigger"
    },
    {
      "parameters": {
        "prompt": "수신된 텍스트를 TIL(Today I Learned) 형식의 마크다운으로 변환해줘."
      },
      "type": "@n8n/n8n-nodes-langchain.agent",
      "name": "AI Agent"
    },
    {
      "parameters": {
        "owner": "GITHUB_USERNAME",
        "repository": "TIL_REPO",
        "filePath": "TIL/{{ $today }}.md",
        "fileContent": "={{ $json.output }}"
      },
      "type": "n8n-nodes-base.github",
      "name": "GitHub Push"
    },
    {
      "parameters": {
        "channelId": "YOUR_DISCORD_CHANNEL_ID",
        "content": "TIL 자동 업로드 및 README 업데이트가 완료되었습니다!"
      },
      "type": "n8n-nodes-base.discord",
      "name": "Discord Send Message"
    }
  ]
}
```

* **Discord Trigger Node**: 특정 채널에 메시지가 생성될 때 워크플로우를 시작함.
* **AI Agent Node**: LLM을 활용하여 단편화된 메모를 구조화된 TIL 마크다운 포맷으로 변환함.
* **GitHub Node**: 생성된 마크다운 텍스트를 파일화하여 커밋 및 푸시하며, 이후 README.md 파일의 내용을 가져와 업데이트하는 과정을 처리함.
* **Discord Node**: 워크플로우의 성공적인 종료를 사용자에게 알림.

### Insight

* n8n을 활용한 워크플로우 자동화의 핵심은 각 노드(Node)의 입력(Input)과 출력(Output) 데이터 구조를 명확히 이해하는 것임.
* 이전 노드에서 어떤 JSON 데이터를 넘겨주고, 현재 노드에서 어떤 매개변수(Parameter)를 필요로 하는지만 파악하면 복잡해 보이는 자동화 로직도 직관적으로 조립할 수 있음.
* 이러한 노드 기반 시각적 프로그래밍(Visual Programming)은 개발의 진입장벽을 크게 낮춰주며, 실질적인 비즈니스 로직이나 프롬프트 엔지니어링에 더 집중할 수 있게 해줌. (훌륭한 인사이트를 제공해준 후츠릿에게 감사함)
