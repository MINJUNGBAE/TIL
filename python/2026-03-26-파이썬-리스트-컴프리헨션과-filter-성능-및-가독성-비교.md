### Context
파이썬(Python)에서 리스트 내포(List Comprehension)를 작성할 때 다중 조건문(`if`)을 중첩하면 가독성이 저하되는 문제가 발생합니다.
이를 해결하기 위해 내장 함수인 `filter()` 사용을 고려할 수 있지만, 대규모 데이터 처리 시 두 방식 간의 유의미한 성능 차이가 존재하여 어떤 방식을 선택할지 고민이 필요합니다.

### Core
다중 조건을 처리하는 두 가지 방식과 성능 차이의 원인은 다음과 같습니다.

* **리스트 내포(List Comprehension)의 다중 조건**
  * 리스트 내포 안에서 여러 개의 `if`를 연속해서 쓰면 암묵적으로 `and` 연산으로 처리됩니다.
  * C 언어 수준에서 최적화되어 실행되므로 루프 처리 속도가 매우 빠릅니다.
  ```python
  data = range(100000)
  # 두 조건을 만족하는 데이터 필터링
  result_comp = [x for x in data if x % 2 == 0 if x % 3 == 0]
  ```

* **filter() 함수와 lambda의 조합**
  * `filter()`는 함수형 프로그래밍 스타일을 제공하여 의도를 명확히 할 수 있습니다.
  * 하지만 사용자 정의 함수나 `lambda`를 사용할 경우, 파이썬 인터프리터에서 매 요소마다 함수 호출 오버헤드(Function Call Overhead)가 발생하여 성능이 크게 저하됩니다.
  ```python
  data = range(100000)
  # 람다 함수 호출로 인한 오버헤드 발생
  result_filter = list(filter(lambda x: x % 2 == 0 and x % 3 == 0, data))
  ```

### Insight
대규모 데이터를 처리할 때 리스트 내포가 `filter()`와 `lambda` 조합보다 압도적으로 우수한 성능을 보여줍니다.
하지만 팀의 구성원과 프로젝트의 유지보수성을 고려하여 상황에 맞는 선택이 필요합니다.

* 파이썬 초심자나 비전공자가 포함된 팀이라면 복잡한 리스트 내포보다는 직관적인 `filter()`나 명시적인 제어문을 사용하여 가독성을 높이는 것이 유리할 수 있습니다.
* 반면 대용량 데이터 파이프라인이나 성능이 중요한 구간(Performance-Critical Section)에서는 반드시 리스트 내포를 사용하여 함수 호출 오버헤드를 최소화해야 합니다.
* 만약 조건이 너무 복잡하여 리스트 내포의 가독성이 심각하게 훼손된다면, 조건 판별 로직을 별도의 일반 함수로 분리하는 방법도 고려할 수 있습니다.
